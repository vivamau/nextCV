# Session: GlobalJobs.org Scraper Fix

## Date
2026-03-25

## Task
Fix the scraper for https://globaljobs.org/ which was returning empty content.

## Context
User reported that the scraper for globaljobs.org returns empty content. The scraping system has three main approaches:
1. Workday API - Direct JSON API calls for `myworkdaysite.com` domains
2. Taleo (.ftl) - Cheerio-based extraction for Taleo job boards
3. Generic AI Parser - Uses AI to extract jobs from cleaned HTML text

## Investigation Steps

### 1. Analyzed Scraper Architecture
- Reviewed [`backend/services/scraperService.js`](backend/services/scraperService.js) - Puppeteer-based dynamic scraper
- Reviewed [`backend/routes/jobRoutes.js`](backend/routes/jobRoutes.js) - Main scraping logic
- Reviewed [`backend/services/aiService.js`](backend/services/aiService.js) - AI-based job listing parser

### 2. Created Debug Scripts
- [`backend/tests/test_globaljobs_debug.js`](backend/tests/test_globaljobs_debug.js) - Tests both axios and Puppeteer scraping
- [`backend/tests/test_globaljobs_simple.js`](backend/tests/test_globaljobs_simple.js) - Simple test with smaller sample
- [`backend/tests/test_globaljobs_fix.js`](backend/tests/test_globaljobs_fix.js) - Final verification test

### 3. Ran Debug Tests
Results from `test_globaljobs_debug.js`:
- Static fetch with axios: ✅ Success (458,644 bytes)
- Dynamic fetch with Puppeteer: ✅ Success (449,328 bytes)
- AI Parsing: ❌ Failed - "Empty response from Ollama flawlessly"

### 4. Root Cause Analysis
The issue was NOT with scraping - both axios and Puppeteer successfully fetched content from globaljobs.org.

The issue was with AI PARSING:
- When sending 15,000+ characters to the AI model (glm-5:cloud), it returned empty responses
- When sending only 2,743 characters, the AI successfully parsed 19 jobs
- The 15,000 character limit in [`backend/routes/jobRoutes.js:186`](backend/routes/jobRoutes.js:186) was too large for the AI model

## Solution Implemented

Reduced the cleaned text limit from **15,000 to 8,000 characters** in [`backend/routes/jobRoutes.js:186`](backend/routes/jobRoutes.js:186):

```javascript
// Before
.substring(0, 15000); // 15,000 chars is fully safe with expanded num_ctx flawlessly

// After
.substring(0, 8000); // 8,000 chars is safer for AI models to avoid empty responses
```

## Lessons Learned
1. AI models can be overwhelmed by large context sizes, even if they theoretically support them
2. When debugging scraper issues, test each component independently (fetching vs. parsing)
3. Reducing context size can improve AI parsing reliability without significant loss of information
4. Always test with smaller samples first to isolate the issue
